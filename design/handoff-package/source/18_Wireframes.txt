This document translates the IA and UX rules into concrete screen concepts. The wireframes are intentionally mid-fidelity: they define hierarchy, zones, and behavioral responsibilities without pretending that the final visual design is settled.

# 1. Wireframing objectives
Lock screen hierarchy before visual polish begins.
Verify that the product can expose recommendation, evidence, risk, and action in one coherent canvas.
Ensure that mobile and operator surfaces inherit the same mental model.

# 3. Main decision workspace concept
The main decision workspace is the most important screen in the product. The recommendation hero, risk summary, and disagreement block are above the fold. Evidence narrative and challenge controls occupy the center. The right context pane is never decorative; it is a working surface for risk, confidence, provenance, and action.

# 4. Engine comparison concept
Comparison should not feel like a separate product. It answers one question: how do multiple analytical engines differ, and what synthesis should the user trust?

Figure 18-2. Engine comparison workspace wireframe.
Use a structured matrix for direct comparison across stance, confidence, risk, and notes.
Supplement the matrix with one visual explaining where alignment is high or low.
Finish with a synthesis panel that guides the user back into the decision workspace.

# 5. Mobile decision concept
Mobile supports triage, monitoring, and lightweight challenge. It should surface a concise recommendation card first, then stack evidence and risk in digestible modules.

# 6. Admin and ops concept

# 9. Engineering handoff annotations
The wireframes should be read as layout and behavior contracts rather than visual suggestions only. During implementation the following annotations must stay attached to each core screen concept.
Annotation type
Examples
Why it must exist

# 11. Screen concept sign-off criteria
A designer and engineer should be able to agree on component boundaries from the wireframes alone.
A product reviewer should be able to point to the primary action on every concept screen without guessing.
Every concept should make degraded and empty states plausible, even if not all are drawn directly on the wireframe.
