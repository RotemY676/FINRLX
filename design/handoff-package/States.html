What each state means

Canonical definitions — use these to decide which state your component is in at any moment.

Default
 All inputs present, freshness within SLO, user has full permissions. Render the full component as designed.
Loading
 Waiting on data > 300ms. Show skeleton that mirrors final layout. Never show a spinner in the middle of a layout block — it should already look like the thing it's becoming.
Empty
 No data exists — this is a state of the world, not a failure. Explain why it's empty and offer the next action. Tone is neutral or positive (e.g. "queue clear").
Degraded
 Component renders but at reduced fidelity — stale input, capped confidence, one engine missing. Show the banner inline, never hide the degradation. User should still be able to proceed with care.
Error
 Component cannot render at all — service exception, timeout, data corruption. Name what failed in plain language, expose an incident ID if one exists, and offer a retry or fallback (e.g. cached).
Permission‑limited
 User lacks the role or entitlement for some or all of the data. Render what they can see, mask the rest with 🔒, and link to request access. Never silently hide — the gap must be visible.
