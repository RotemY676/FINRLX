# SIMPLE MODE COPY DECK (Program LEAP S1.3)

Binding strings for S5/S6. Register: plain verbs, sentence case, no filler,
no selling. Every automatic conclusion carries its evidence path; nothing
implies advice, prediction, or assurance. Validated against the S9
advice-language pattern set (see wording test note at bottom).

## Hero (J0)
- H1: `Research any stock`
- Input placeholder: `Try NVDA`
- Primary action: `Research`
- Subline: `Automatic 360° research: prices, news, technicals, and a model
  tournament — with the evidence for every conclusion.`

## Progress (J1)
- Stage names: `Fetching price history` · `Reading recent news` ·
  `Computing technical signals` · `Running the model tournament` ·
  `Assembling your dossier`
- Long-wait line (>20s): `First-time research for a ticker takes longer;
  results are cached afterward.`

## Dossier (J2)
- Stance chip labels: `constructive` / `neutral` / `cautious`
  - **Mapping from payload (binding):** `buy→constructive`, `hold→neutral`,
    `sell→cautious`, `trim→cautious`. The payload's raw stance words are
    engine vocabulary for Pro surfaces and never render in Simple Mode; the
    S5 wording test enforces this.
  - Hover/long-press label: `Research stance from the engine ensemble — not advice.`
- Regime chip: payload label verbatim (`uptrend` / `downtrend` / `risk-off` /
  `neutral`), hover: `Rule-based research overlay, not a prediction.`
- Freshness: `Data through {date}`
  - Stale chip (2–5 sessions): `Data through {date}`  (caution-soft)
  - Degraded banner (>5): `Price data ends {date}; conclusions may be outdated.`
- Card titles: `Technical` · `News & sentiment` · `Fundamentals` · `Model insight`
- Technical drill link: `All signals`
- News annotation prefix: `Why it matters:` (+ meta line `Generated by {model} ·
  based on item from {date}`)
- News degraded note: payload note verbatim (`News source unavailable — section
  degraded, analysis continued without it.`)
- Fundamentals note: payload note verbatim.
- Model insight winner line: `{name} — validation score {score}, chosen over
  {n} candidates after walk-forward validation with overfitting penalties.`
- Model kind chips: `rule-based` / `machine-learning`
- Scoreboard link: `How this was chosen`
- Scoreboard column heads: `Candidate · Kind · Train Sharpe · Validation Sharpe ·
  Divergence · Penalty · Score`
- RL leg (verbatim from payload when queued): status note as returned; heading:
  `Reinforcement-learning candidates`
- Insufficient history: `The tournament needs more history to validate
  candidates honestly.` (payload rationale verbatim)
- Actions: `Compare` · `Export`
- Disclaimer strip (always, payload verbatim): `Research analysis, not
  investment advice.` · `Walk-forward validation on historical data; past
  performance does not predict future results.`

## Compare (J3)
- Title: `Compare`
- Add flow: `Add a ticker (up to 4)`
- Divergence marker tooltip: `Tickers differ on this dimension — values shown
  are measured, side by side.`
- Per-column failure: `Couldn't build this dossier: {reason}`
- Footer: `Comparison dimensions come from each ticker's dossier; see the
  per-ticker dossier for full evidence and limitations.`

## Empty & error (J4/J5)
- No data: `No price data found for "{input}".` + `Did you mean:` suggestions
- API error: `Research is temporarily unavailable. Your ticker is kept —
  retry when ready.` · action `Retry {ticker}`

## Export
- Exported HTML footer: the two disclaimer strings verbatim + `Data through
  {date}` + `Model chosen by walk-forward tournament — penalties included in
  the embedded scoreboard.`

## Words this product never uses in Simple Mode
buy · sell · should · must · guaranteed · will outperform · opportunity ·
signal to act · act now · sure · can't lose — plus anything in the S9
`_ADVICE_PATTERNS` set. Enforcement: S5 adds a wording test asserting these
strings are absent from Simple Mode components (mirrors the 7F/8F safe-
language test pattern).
